# REMOVER — План и архитектура

> Бесплатный браузерный удалитель фона профессионального уровня.
> Работает локально на компьютере и одним и тем же кодом публикуется как бесплатный сайт.

---

## 1. Цель проекта (одной строкой)

Загрузил картинку → через пару секунд получил PNG с прозрачным фоном **без каёмки**, в **полном разрешении**, для **любой** картинки. Бесплатно и приватно.

---

## 2. Ключевые принципы (не меняем без веской причины)

1. **Всё считается в браузере** — нет сервера, нет затрат, картинки не покидают устройство.
2. **Бесплатно навсегда** — только свободные (MIT/Apache) движки и бесплатный хостинг.
3. **Сохраняем оригинал** — нейросеть считает маску, а накладываем её на исходник в полном разрешении. Качество не теряется.
4. **Matting, а не вырезание** — мягкая прозрачность (волосы, кружево, веточки).
5. **Своя деконтаминация края** — автоматически вычищаем чужой цвет фона из полупрозрачных пикселей. Это убивает жёлтую/белую каёмку. Наш главный секрет.
6. **Один код — два применения**: `npm run dev` локально и деплой на бесплатный хостинг — без переписывания.

---

## 3. Архитектура (простыми словами)

```
┌─────────────────────────────────────────────────────────┐
│  Браузер пользователя (всё происходит здесь)             │
│                                                           │
│  [Загрузка картинки]                                      │
│         │                                                 │
│         ▼                                                 │
│  ДВИЖОК-МОЗГ  →  считает маску (кто объект, кто фон)      │
│  @imgly (быстрый)  /  BiRefNet (качественный, про-режим) │
│         │                                                 │
│         ▼                                                 │
│  НАШ СЛОЙ ОБРАБОТКИ КРАЯ                                  │
│  • деконтаминация (убрать цвет фона с края)              │
│  • мягкость края (feather)                               │
│  • наложение маски на ОРИГИНАЛ (полное разрешение)       │
│         │                                                 │
│         ▼                                                 │
│  [Предпросмотр на шахматке/цвете] → [Скачать PNG]        │
└─────────────────────────────────────────────────────────┘
```

**Стек:** Vite + чистый JavaScript (без лишних фреймворков), Canvas API для обработки пикселей,
`@imgly/background-removal` как основа, `@huggingface/transformers` (BiRefNet + WebGPU) для про-режима.

## 4. Фазы разработки

Каждая фаза — самостоятельный, работающий результат. Не начинаем следующую, пока текущая не работает и не закоммичена.

### Фаза 0 — Каркас
- Инициализация Vite-проекта, структура папок.
- Проект запускается локально (`npm run dev`), открывается пустая страница.
- `git init`, первый коммит, приватный бэкап на GitHub.
- **Готово, когда:** страница открывается локально, код на GitHub.

### Фаза 1 — MVP: базовое вырезание
- Перетащил/выбрал картинку.
- Движок `@imgly` считает маску, фон удаляется.
- Результат виден на «шахматке» (прозрачность).
- Кнопка «Скачать PNG» — в полном разрешении.
- **Готово, когда:** любую картинку можно вырезать и скачать PNG.

### Фаза 2 — Качество края (наш «вау»)
- Деконтаминация края: убираем цветную каёмку.
- Ползунок «мягкость края» (feather).
- Ползунок «сила очистки каёмки».
- Наложение маски строго на оригинал в полном разрешении.
- **Готово, когда:** на примере с жёлтым фоном каёмки нет, край мягкий.

### Фаза 3 — UX и предпросмотр (превращаем в «вау»)
- Красивый, современный дизайн (аккуратный, приятный).
- Предпросмотр результата на разных фонах (белый/чёрный/цвет/своя картинка).
- Ползунок «до/после».
- Индикатор прогресса, обработка нескольких файлов подряд.
- **Готово, когда:** пользоваться приятно, результат сразу видно на нужном фоне.

### Фаза 4 — Про-режим для сложных кадров
- Подключаем BiRefNet через transformers.js + WebGPU.
- Переключатель «Быстро / Максимальное качество».
- Поддержка более высокого разрешения на мощных устройствах.
- **Готово, когда:** сложные кадры (волосы, кружево) заметно чище, чем в быстром режиме.

### Фаза 5 — Публикация и полировка
- Бесплатный деплой (GitHub Pages / Cloudflare Pages) — публичный сайт.
- Возможна работа офлайн (PWA) — движок кэшируется.
- Финальная вычитка, README, инструкция для новичка.
- **Готово, когда:** сайт открывается по ссылке у любого, работает без затрат.

## 5. Что НЕ делаем (границы, чтобы не распыляться)

- ❌ Свой сервер / облако / оплата за обработку.
- ❌ Обучение своей нейросети с нуля.
- ❌ Регистрация, аккаунты, база данных, платежи.
- ❌ Мобильное приложение в сторах (сайт открывается и на телефоне).
- ❌ Ручные инструменты уровня Photoshop (кисти, слои) — только автоматическое вырезание + тонкая подстройка ползунками.

---

## 6. Риски и как их закрываем

| Риск | Решение |
|---|---|
| Слабое устройство тормозит | Быстрый режим по умолчанию; про-режим включается вручную |
| WebGPU есть не везде | Автооткат на WASM/CPU |
| Очень большое разрешение | Обработка маски в уменьшенном виде, наложение — на полный оригинал |
| Каёмка на сложном фоне | Наш слой деконтаминации + ползунки |
