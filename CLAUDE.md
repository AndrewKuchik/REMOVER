# REMOVER — рабочие правила проекта

> Этот файл читается автоматически. Он держит нас в фокусе. Перед работой — свериться с ним,
> не переанализировать проект заново, идти только в нужном направлении.

## Что это за проект
Бесплатный **браузерный** удалитель фона. Всё считается на устройстве пользователя, без сервера.
Полное описание и фазы — в [PLAN.md](PLAN.md). Итоги работы — в [ОТЧЁТ.md](ОТЧЁТ.md).

## Решения приняты — НЕ пересматривать без явной просьбы
- **Где считаем:** только в браузере. Никакого сервера, оплаты, аккаунтов, БД.
- **Стек:** Vite + чистый JavaScript + Canvas API.
- **Движок (ГИБРИД, прорывной путь):**
  - ПО УМОЛЧАНИЮ — **мгновенный алгоритмический кейер** (`keyer.js`, 0 МБ, без загрузки): авто-определение фона по полосе краёв (Lab, отбраковка контаминации) + заливка с запечатыванием разрывов (dilate) + мягкая альфа. Идеален для рисунков/логотипов/товаров/сканов на светлом фоне. Считает на прокси ≤1024, маска — на оригинал.
  - ОПЦИЯ «🧠 Обработать нейросетью» — RMBG-1.4 q8, **грузится ЛЕНИВО** (`router.js` → dynamic import `engine.js`), поэтому transformers.js НЕ в стартовом бандле (first-paint ~14 КБ, мгновенный старт).
  - `router.js` — фасад: `prepareInstant` / `prepareNeural` / `keyFromProxy` (живой ползунок чувствительности).
  - Позиционирование: «мгновенный приватный удалитель фона для иллюстраций, а не селфи».
- **Лицензия RMBG-1.4:** CC BY-NC (только некоммерческое). Для прототипа ок; для коммерции — купить BRIA или сменить модель. ВАЖНО помнить.
- **Главная фишка:** свой слой деконтаминации края (`refine.js`) — убираем цветную каёмку + мягкость.
- **Разрешение:** маску считаем в 1024², но накладываем на ОРИГИНАЛ в полном разрешении.
- **UX:** спокойная загрузка (полоса + МБ, «один раз»), обработка со счётчиком секунд, мягкое уведомление вместо пугающих ошибок. Никаких тревожных иконок.
- **Хостинг:** GitHub Pages (бесплатно). Смена хостинга качество не даёт; бесплатные серверные функции модель не тянут — проверено разведкой.

## Границы — этого НЕ делаем
Свой сервер · обучение сети с нуля · регистрация/платежи/БД · приложение в сторах ·
ручные кисти/слои как в фотошопе.

## Как работаем
- Идём строго по фазам из PLAN.md. Одна фаза = рабочий результат + коммит.
- **Статус:** ВСЕ ФАЗЫ 0–5 ГОТОВЫ И ОПУБЛИКОВАНЫ. Живой сайт: https://andrewkuchik.github.io/REMOVER/
- Репозиторий публичный, GitHub Pages включён, автодеплой при каждом пуше в main. Работоспособность подтверждена на живом URL.
- Коммитить по ходу сам, коротко, по-русски. Бэкап на GitHub (приватный репо, аккаунт AndrewKuchik).
- С пользователем — по-русски, простыми словами, он новичок в VS Code и git. Минимум ручных шагов.
- После значимого этапа — короткая запись в [ОТЧЁТ.md](ОТЧЁТ.md).
