# REMOVER — рабочие правила проекта

> Этот файл читается автоматически. Он держит нас в фокусе. Перед работой — свериться с ним,
> не переанализировать проект заново, идти только в нужном направлении.

## Что это за проект
Бесплатный **браузерный** удалитель фона. Всё считается на устройстве пользователя, без сервера.
Полное описание и фазы — в [PLAN.md](PLAN.md). Итоги работы — в [ОТЧЁТ.md](ОТЧЁТ.md).

## Решения приняты — НЕ пересматривать без явной просьбы
- **Где считаем:** только в браузере. Никакого сервера, оплаты, аккаунтов, БД.
- **Стек:** Vite + чистый JavaScript + Canvas API.
- **Движок-основа:** `@imgly/background-removal` (быстрый режим).
- **Движок про-режима:** `@huggingface/transformers` → BiRefNet + WebGPU.
- **Главная фишка:** свой слой деконтаминации края (убираем цветную каёмку) + мягкость края.
- **Разрешение:** маску считаем как удобно движку, но накладываем на ОРИГИНАЛ в полном разрешении.
- **Хостинг потом:** бесплатный (GitHub Pages / Cloudflare Pages), тем же кодом.

## Границы — этого НЕ делаем
Свой сервер · обучение сети с нуля · регистрация/платежи/БД · приложение в сторах ·
ручные кисти/слои как в фотошопе.

## Как работаем
- Идём строго по фазам из PLAN.md. Одна фаза = рабочий результат + коммит.
- **Текущая фаза:** Фаза 2 (качество края). Фазы 0 и 1 готовы и проверены в реальном браузере.
- Коммитить по ходу сам, коротко, по-русски. Бэкап на GitHub (приватный репо, аккаунт AndrewKuchik).
- С пользователем — по-русски, простыми словами, он новичок в VS Code и git. Минимум ручных шагов.
- После значимого этапа — короткая запись в [ОТЧЁТ.md](ОТЧЁТ.md).
